# Рабочий бот: Dadata + Checko + Telegram + Replit

## Что собрал
- управление ботом выполняется через Telegram-интерфейс (polling-бот);
- web-панель используется только для первичной настройки токенов и API-ключей;
- разделение сущностей: `company`, `entrepreneur`, `person`;
- Dadata для поиска по ИНН, ОГРН, ОГРНИП, названию и ФИО ИП;
- Checko для расширенных разделов;
- reply-меню в стиле EGRUL_BOT;
- хранение конфигурации и логов в локальном JSON-файле, без обязательной PostgreSQL.

## Что умеет
### ЮЛ / ООО
- Реквизиты
- Финансы
- Налоги
- Закупки
- Проверки
- Арбитраж
- Исполнительные производства
- Лицензии
- История изменений
- Риски
- Дополнительно

### ИП
- Реквизиты ИП
- Финансы (честно сообщает, что /finances не дает бухотчетность для ИП)
- Налоги
- Закупки
- Проверки
- Арбитраж
- Исп. производства (честно сообщает, что метод Checko для организаций)
- Лицензии
- Текущая ФНС
- ЕФРСБ
- Риски
- Дополнительно

### Физлицо
- Основные сведения
- ЕФРСБ
- Санкции
- Товарные знаки
- Связанные ИП и ЮЛ
- Риски

## Что важно
- Бот управляется через Telegram; web-панель нужна только для ввода/обновления ключей и токенов.
- Настройки и логи пишутся в `.data/bot-storage.json`.
- База данных не обязательна.
- `Checko API key` нужен для всех расширенных разделов.
- Без Checko бот все равно ищет организации и ИП через Dadata.

## Запуск
```bash
npm install
npm run dev
```

## Первичная настройка через web-панель
- Telegram token
- Dadata API key
- Dadata secret key — опционально
- Checko API key — для расширенных разделов

## Сборка рабочего бота из архива

В репозитории хранится архив `Dadata-Bot-working.zip` с готовым проектом бота.
Собрать рабочую директорию можно так:

```bash
python scripts/assemble_working_bot.py --force
cd working-bot
npm install
npm run dev
```

Скрипт валидирует, что в распакованном проекте есть ключевые файлы бота (`package.json`, `server/index.ts`, `server/routes.ts`, `client/src/main.tsx`).
Также при сборке автоматически применяется совместимый патч форматирования раздела «Дополнительно» для полей Checko (ПоддержМСП, СЧР, ЕФРСБ, РНП, санкции и связанные риск-признаки).

## Что просмотрено из архивов
- `Dadata-Bot.zip` — исходная база Replit-проекта
- `Dadata-Bot-checko-patch.zip` — промежуточный патч
- `Чеко.zip` — PDF-документация Checko API + справочники
- `Бот1000.zip` — сборный архив с дублями проекта, документацией, скринами и справочниками

## Что не использовал в рантайме
- SQL-справочники ОКВЭД/ОКПД/ОКФС/ОКОПФ
- Excel-справочники статусов СЮЛСТ/СИПСТ
- `format.rar` с XSD/DOC-форматами

## Замечания
- `format.rar` распознан и просмотрен по списку файлов, но не распакован автоматом в контейнере из-за отсутствия rar-экстрактора.
- Для запроса по 12-значному ИНН бот сначала пытается определить ИП, затем физлицо.
