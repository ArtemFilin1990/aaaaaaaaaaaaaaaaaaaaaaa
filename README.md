# Telegram-бот проверки компаний (Checko.ru)

Telegram-бот на Python 3.11 + aiogram для проверки компаний по ИНН через [API Checko.ru](https://checko.ru/integration/api).

Показывает карточку компании с inline-кнопками для получения финансов, арбитражных дел, госзакупок и связей — без повторных API-запросов (кэш на сессию).

---

## Структура проекта

```
.
├── bot/
│   ├── main.py          # Точка входа: aiogram-бот, хэндлеры, кэш
│   ├── checko_api.py    # Клиент Checko.ru API + функции форматирования
│   └── keyboards.py     # Inline-клавиатура карточки компании
├── .env.example         # Шаблон переменных окружения
├── requirements.txt     # Зависимости
└── README.md
```

---

## Быстрый старт

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Создать `.env`

```bash
cp .env.example .env
```

Заполнить `.env`:

```env
TELEGRAM_TOKEN=<токен от @BotFather>
CHECKO_API_KEY=<ключ из личного кабинета checko.ru>
```

### 3. Запустить бота

```bash
cd bot
python main.py
```

## Использование

| Действие | Результат |
|---|---|
| `/start` | Приветствие с инструкцией |
| Отправить ИНН (10 или 12 цифр) | Карточка компании + inline-кнопки |
| 📊 Финансы | Выручка, прибыль от продаж, чистая прибыль (последние 3 года) |
| ⚖️ Суды | Арбитражные дела (номер, дата, суд, сумма иска, ссылка КАД) |
| 💰 Госзакупки | Контракты по 44-ФЗ (номер, заказчик, предмет, цена, ссылка ЕИС) |
| 🔗 Связи | Руководители и учредители компании |
| 📋 Полный отчёт | JSON-ответ `/v2/company` |

## API Checko.ru — используемые эндпоинты

Все запросы — GET `https://api.checko.ru/v2/...?key=API_KEY&inn=INN`

### `/v2/company` — ЕГРЮЛ

Карточка компании + связи. Поля ответа (`data`):

| Поле | Описание |
|---|---|
| `НаимПолн` / `НаимСокр` | Полное / сокращённое наименование |
| `ИНН` | ИНН |
| `ОГРН` | ОГРН |
| `КПП` | КПП |
| `Статус` | Статус (Действующее / Ликвидировано / …) |
| `ДатаРег` | Дата государственной регистрации |
| `ДатаЛикв` | Дата ликвидации |
| `РегионКод` | Код региона РФ |
| `ЮрАдрес` | Юридический адрес |
| `ОКВЭД` | Основной вид деятельности |
| `Руководители[]` | Список руководителей: `ФИО`, `Должн` |
| `Учредители[]` | Список учредителей: `НаимПолн`/`ФИО`, `ИНН`, `ДоляПроцент` |

### `/v2/finances` — Финансовая отчётность

Параметры: `inn` или `ogrn`.

Ответ: `data.{год}.{код_строки}` = значение в тыс. руб.

Ключевые строки:

| Код | Показатель |
|---|---|
| `2110` | Выручка |
| `2120` | Себестоимость |
| `2200` | Прибыль от продаж |
| `2300` | Прибыль до налогообложения |
| `2400` | Чистая прибыль |

### `/v2/legal-cases` — Арбитражные дела

Параметры: `inn`/`ogrn`, `role` (plaintiff/defendant), `sort`, `page`, `limit`.

Поля записи (`data.Записи[]`):

| Поле | Описание |
|---|---|
| `Номер` | Номер дела |
| `Дата` | Дата возбуждения |
| `Суд` | Наименование суда |
| `СуммИск` | Сумма исковых требований |
| `СтрКАД` | Ссылка на Картотеку арбитражных дел |
| `Ист[]` | Истцы: `ИНН`, `Наим`, `Адрес` |
| `Ответ[]` | Ответчики: `ИНН`, `Наим`, `Адрес` |

### `/v2/contracts` — Госзакупки

Параметры: `inn`/`ogrn`, `law` (44/94/223), `role` (customer/supplier), `sort`, `page`.

> С 2019 года ЕИС не публикует данные поставщиков по 223-ФЗ.

Поля записи (`data.Записи[]`):

| Поле | Описание |
|---|---|
| `РегНомер` | Регистрационный номер |
| `Дата` | Дата подписания |
| `ДатаИсп` | Дата исполнения |
| `Цена` | Цена контракта, руб. |
| `СтрЕИС` | Ссылка на zakupki.gov.ru |
| `Заказ{}` | Заказчик: `НаимСокр`, `ИНН` |
| `Постав[]` | Поставщики: `НаимСокр`/`ФИО`, `ИНН` |
| `Объекты[]` | Предметы закупки: `Наим`, `ОКПД2` |

## Кэширование

Каждый эндпоинт имеет отдельный словарь-кэш (`dict[inn, response]`) в памяти процесса. При повторных нажатиях кнопок API не вызывается — данные берутся из кэша. Кэш живёт до перезапуска бота.
